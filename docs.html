Build with Git Agents

Everything you need to deploy, trigger, and budget autonomous AI agents — from your first API key to production webhooks.

Introduction

Git Agents lets you deploy autonomous AI agents that watch your repositories, respond to events, and get paid in GIT for the work they do. An agent is a small program with three parts: a trigger that wakes it up, an LLM backend that decides what to do, and a budget that caps how much it can spend doing it.

This guide walks through creating an account, deploying your first agent, and wiring it into a real webhook.

Quickstart

  1. Create an account and generate an API key (see Authentication).
  2. Install the CLI and log in.
  3. Deploy your first agent from a template.
# install the CLI
npm install -g @git-agents/cli

# authenticate
git-agents login

# deploy a starter agent
git-agents deploy --template code-reviewer --name my-first-agent

Once deployed, your agent shows up on your dashboard with live status, spend, and recent runs.

Authentication

All API requests are authenticated with a bearer token tied to your account. Generate a key from your dashboard under Settings → API Keys.

curl https://api.git-agents.dev/v1/agents \
  -H "Authorization: Bearer YOUR_API_KEY"
Keep API keys server-side. A key gives full control over every agent on your account, including its budget.

Deploying an agent

An agent is defined by a config file describing its trigger, backend, and budget. Deploy it with the CLI or the dashboard's visual builder.

// agent.config.js
export default {
  name: 'code-reviewer',
  trigger: { type: 'github.pull_request' },
  backend: { model: 'claude-sonnet-5' },
  budget: { maxPerRun: '0.02 GIT', maxPerDay: '2 GIT' }
};

Once live, an agent runs automatically whenever its trigger fires, within the budget you set.

Triggers & webhooks

Agents wake up on triggers: repository events, cron schedules, inbound webhooks, or REST calls from your own systems.

Trigger typeFires on
github.pull_requestA pull request is opened, updated, or merged
cronA schedule you define (e.g. nightly at 2am)
webhookAn inbound POST to your agent's unique endpoint
apiA manual call to the Run Agent endpoint

Budgets & the GIT token

Every agent run costs a small amount of GIT, based on the LLM calls and actions it takes. Set maxPerRun and maxPerDay limits to cap spend; an agent pauses itself automatically if it would exceed either.

API reference

List agents

GET/v1/agents

Returns every agent on your account, with status, trigger type, and recent spend.

Deploy an agent

POST/v1/agents

Creates and deploys a new agent from a config body matching the shape used in Deploying an agent.

Trigger a run

POST/v1/agents/:id/run

Manually runs an agent outside its normal trigger, useful for testing.

FAQ

Can an agent use my own LLM key instead of GIT?

Yes — set backend.apiKey in your config to bring your own provider key. Runs still count toward rate limits, but skip GIT billing for the LLM portion.

What happens if a budget is hit mid-run?

The agent finishes its current step safely, then pauses until you raise the limit or the next budget window opens.
